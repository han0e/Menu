# 디자이너별 개별 메뉴판 아키텍처 기획안

## 1. 개요 및 구조 결정
디자이너별 특화 메뉴가 존재하고, 커트 등 특정 시술 가격이 다르다는 요구사항에 따라 **[완전 독립형 데이터 구조 + 템플릿(공본) 복사 기능]**을 채택합니다.

이 방식은 각 디자이너가 자신만의 완전히 독립적인 메뉴와 카테고리를 가지게 하되, 초기 세팅의 번거로움을 줄이기 위해 '기본 메뉴'를 한 번에 복사해 오는 기능을 제공하는 하이브리드 방식입니다.

> 💡 **원장님 질문에 대한 답변 (공통메뉴 수정 시 다른 디자이너에게 영향을 주나요?)**
> **전혀 영향을 주지 않습니다.** '공통 메뉴(공본)'는 말 그대로 최초 세팅을 위해 **복사해오는 원본(템플릿)** 역할만 합니다. 디자이너가 공통 메뉴를 내 메뉴판으로 '복사'해 온 이후부터는 완전히 내 소유의 독립적인 데이터가 됩니다. 따라서 **내 메뉴판에서 가격이나 설명을 마음대로 수정해도 다른 디자이너의 메뉴나 공통 메뉴 원본에는 아무런 영향을 주지 않습니다.**

### 장점
- **완전한 독립성**: 디자이너 간의 메뉴 데이터가 엉키거나 충돌할 일이 없습니다. 내가 설명을 바꾸거나 가격을 바꿔도 나에게만 적용됩니다.
- 각자 본인만의 특화 메뉴 추가나 가격 변경이 매우 자유롭습니다.
- 향후 기능 확장이 쉽고, 프론트엔드 유지보수(코드)가 단순합니다.
- DB 부하나 테이블 증가 이슈가 전혀 없습니다. (인덱스를 통해 1만명의 디자이너가 있어도 즉시 검색됨)

## 2. 데이터베이스 (Supabase) 변경 계획

디자이너별 구분을 위해 기존 메뉴 테이블들에 `designer_id` 컬럼을 추가해야 합니다.
`designer_id`가 `null`인 데이터는 모든 디자이너가 복사해갈 수 있는 **'공본(Template)'**으로 취급합니다.

아래 SQL을 Supabase SQL Editor에서 실행하게 될 예정입니다.

```sql
-- 1. categories 테이블에 designer_id 컬럼 추가 (auth.users 참조)
ALTER TABLE public.categories 
ADD COLUMN designer_id uuid REFERENCES auth.users(id) ON DELETE CASCADE;

-- 빠른 조회를 위한 인덱스 생성
CREATE INDEX idx_categories_designer_id ON public.categories(designer_id);

-- 2. menu_items 테이블에 designer_id 컬럼 추가 (auth.users 참조)
ALTER TABLE public.menu_items 
ADD COLUMN designer_id uuid REFERENCES auth.users(id) ON DELETE CASCADE;

-- 빠른 조회를 위한 인덱스 생성
CREATE INDEX idx_menu_items_designer_id ON public.menu_items(designer_id);
```

## 3. 프론트엔드 (React) 변경 계획

### `src/pages/MenuAdmin.jsx` (관리자 페이지)
1. **내 메뉴만 보기**: 기존 `supabase.from('menu_items').select('*')` 조회 로직에 `.eq('designer_id', session.user.id)`를 추가합니다.
2. **저장 시 내 ID 부여**: 카테고리 및 메뉴 생성/수정 로직(`saveCategory`, `saveMenu`)에 현재 로그인한 `session.user.id`를 저장하도록 수정합니다.
3. **[신규 기능] 템플릿 가져오기**: 카테고리가 1개도 없는 디자이너가 최초 로그인 시, "기본 메뉴판 세팅하기" 버튼을 노출합니다. 이 버튼을 클릭하면 `designer_id`가 null인 공본(Template) 데이터를 읽어와서 해당 디자이너의 ID로 통째로 복사(Insert)합니다.

### `src/pages/Main.jsx` (고객용 메인 화면)
1. **해당 디자이너 메뉴 렌더링**: 고객이 태블릿으로 메뉴판을 볼 때, 현재 로그인되어 있는 디자이너의 메뉴만 렌더링하도록 `.eq('designer_id', session.user.id)` 필터를 걸어 데이터를 가져옵니다.
2. (선택사항) 우측 상단이나 하단에 현재 로그인된 디자이너의 이름을 표시하여 "OOO 디자이너의 메뉴판" 임을 명시할 수 있습니다.
